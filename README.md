# CLAMS Project Documentation Hub

This repository serves as the central publication hub for all documentation related to the CLAMS project. It aggregates generated HTML documentation from various source repositories and serves them from a single location.

## Hosted Documentation

| Project | Path | Versioned | Source repo |
|---------|------|-----------|-------------|
| Home / hub pages | `docs/home/` | no | this repo (`documentation/`) |
| clams-python | `docs/clams-python/` | no | `clamsproject/clams-python` |
| vocabulary | `docs/vocabulary/` | no | `clamsproject/clams-vocabulary` |
| mmif-python | `docs/mmif-python/` | yes (`docs/mmif-python/<version>/`) | `clamsproject/mmif-python` |

> [!NOTE]
> `docs/vocabulary/` contains legacy version subdirectories (`1.0.0`–`1.1.0`) from when vocabulary was part of the MMIF specification. These are static historical snapshots, not generated by the current build system.

## How It Works

Documentation source (`.rst` files, Python docstrings) lives in each project's repository. Each source repo has a `publish.yml` workflow that, upon release, calls the reusable `sdk-docs.yml` workflow in this repository. The workflow:

1. Checks out the source repo at the release tag.
1. Runs `python3 build-tools/docs.py --build-ver <version> --output-dir _docs` in the source repo.
1. Copies the output into `docs/<project-name>/` (or `docs/<project-name>/<version>/` for versioned projects).
1. Runs `build-tools/inject-header.py` to add the CLAMS hub navigation header across all projects.
1. Commits and pushes to this repository.

This repository does not contain any documentation *source* files (except for the hub home pages), only the final, published HTML artifacts.

### Building Documentation Locally

These instructions are for developers working on the documentation source files located in the `documentation/` directory. `documentation/` contains the source for the main hub pages (e.g., index, team, events) that's called "home" in the integerated publication pages. 

#### Setup

```
pip install -r build-tools/requirements.home.txt
```

#### Build

From the repository root, you build the documentation using `sphinx-build`. This generates the raw HTML files.

```bash
python build-tools/home.py 
```

This will generate the HTML files in the `docs/home` directory. There's `docs/index.html` (the main landing page) that's simply redirected to `docs/home/index.html`.

After building, you can inject the consistent CLAMS hub header into the generated files. This is necessary for the docs to look correct when deployed. The `--base-url` should point to the root of where the docs will be served. For local viewing, you can use a relative path.

```bash
# Example for deployment
python build-tools/inject-header.py docs/home --base-url https://clams.ai
```

This will edit all html files in `docs/home` in-place.

#### View Locally

```bash
# Simple HTTP server
cd docs
python -m http.server 8000
# Then open http://localhost:8000 in your browser
```
